
### reAnimated, gesture-handler

- 리액트 네이티브에서 에니메이션 및 제스쳐를 다루는 패키지
- UI 쓰레드에서 데이터를 관리하도록 훅을 제공
- Javascript 쓰레드에서 데이터를 관리하면 컴포넌트 리렌더링에 영향을 끼쳐서 성능이 악화될 수 있음

### React Navigation

- 리액트 네이티브에서 네비게이션 처리할 때 사용.
- React Web에서 사용하는 React Router랑은 다름.
- React Router는 페이지 이동시 컴포넌트를 완전 제거/생성하지만, React Navigation은 화면 인스턴스를 재활용 하는 경우가 있음.
- 따라서 React Navigation을 사용할 때에는 useEffect를 잘 사용해서 route.params 값을 state에 잘 초기화 해주어야함.

### 네비게이션 (React-Navigation/native)

 - Stack vs Native Stack 
	 - 성능은 Native Stack이 좀 더 좋아서 선호됨. Native Stack으로 잘 안되면 Stack으로 선회
 - App.js에서 NavigationContainer와  네비게이션 객체의 컴포넌트 생성함수로 스크린을 감싼다.
 - 첫번째 스크린이 기본 화면이 된다.
 - 감싸져 있는 스크린들은 prop으로 navigation과 route를 받을 수 있다.
 - 감싸져 있지 않은 컴포넌트라면 useNavigation, useRoute를 사용할 수 있다.

- 여러 내비게이터를 중첩해서 사용할 수 있다.
	- 근데 보통 스택안에 Drawer를 중첩해서 쓰는듯?

### Text

- Text 컴포넌트 안에 Text 컴포넌트를 넣는 것이 가능함. 
- 이를 이용해 다른스타일을 적용해서 강조할 수 있음.
- 그리고 이러한 중첩 텍스트는 부모의 Style에 영향을 받음.(상속)

## 스타일 (StyleSheet)

- TextAlign은 Text나 TextInput 컴포넌트에만 적용됨.
- overflow : hidden -> 자식요소가 부모 영역을 넘어가면 감춤
- 계단식 스타일
	- 어떠한 컴포넌트에서 조금 변형주고 싶을 때, 사용
	- props에 style prop을 넣고, 기존의 style prop을 배열로 가져감
	- style={[base 스타일, 덮어쓰기 할 스타일, ...]}, 배열뒤쪽에 있는 스타일이 우선시됨.

- expo를 사용할 때, 아이콘 사용할 때는 Ionicons
- 플랫폼마다 다른 스타일 , 컴포넌트를 작성할 땐 파일 확장자 명을 .android, .ios로 구분해서 임포트하면 됨, 간단한거면 PlatformAPI 사용
