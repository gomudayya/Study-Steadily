
### 렌더링, 리렌더링, 마운트, 언마운트

- 렌더링
	- 컴포넌트 함수 실행해서 JSX 반환하는 과정
	- 컴포넌트가 화면에 그려질 때마다 발생
- 리렌더링
	- 이미 렌더링된 컴포넌트를 다시 렌더링
	- state/props 변경, 부모 컴포넌트 리렌더링시 발생
	- 컴포넌트 함수만 재실행, 내부 상태는 유지됨
- 마운트
	- 컴포넌트가 DOM에 처음 삽입되는 과정
	- 컴포넌트가 화면에 처음 나타날 때 발생
	- `useEffect(() => {}, [])` 는 처음 마운트시에만 실행
- 언마운트
	- 컴포넌트가 DOM에서 제거되는 과정
	- 컴포넌트가 화면에서 사라질 때 발생
	- useEffect의 cleanup 함수 실행

- **하위 컴포넌트의 생명주기를 상위 컴포넌트의 상태 변화에 따라 같이 가져가고 싶다면?**
	- key-prop에 상태 관련 값을 추가하면 된다.
	- 상태가 변경되면, 상위 컴포넌트 함수는 리렌더링이 발생하고, 
	  하위 컴포넌트 에서는 언마운트 - 마운트가 일어난다.
