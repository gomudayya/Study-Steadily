### Gradle

- 소프트웨어를 빌드, 테스트, 배포하는 과정을 자동화해 주는 오픈 소스 빌드 자동화 도구

- 소스 코드를 **실행 가능한 프로그램**으로 만들고, **라이브러리를 관리**하며, **테스트를 수행**하는 전 과정을 효율적으로 관리해 주는 빌드시스템


### 핵심 개념 및 특징

- **프로젝트(모듈)**
	- 그래들이 빌드하는 소프트웨어의 단위
	- Gradle 프로젝트 전체는 하나의 루트 프로젝트와 여러 개의 서브프로젝트(모듈)로 구성될 수 있다.

- **작업 단위 (Tasks)** 
	- Gradle은 독립적인 작업 단위인 **Task**를 실행하여 빌드를 수행한다. 
	- ex) 소스 코드 컴파일, JAR 파일 생성, 테스트 실행, 문서 생성 등이 각각의 Task 이다.

- **플러그인 (Plugins)**
	- Gradle은 핵심 엔진만 가지고 있으며, 대부분의 기능은 플러그인을 통해 제공 된다.
	- 예시
		- 자바의 경우 java-library 플러그인을 사용
		- 안드로이드 개발은 안드로이드 빌드를 위한 플러그인을 적용
	- 플러그인을 통해 표준 태스크와, 표준 의존성을 주입받는다.

- **의존성 관리 (Dependency Management)**
	- 외부 라이브러리(Maven Central 등)를 자동으로 다운로드하고, 버전 충돌을 해결
	- 라이브러리를 가져올 장소(예: Maven Central, Google)를 지정 가능
	- 의존성의 사용 범위(Scope)를 정의
		- `implementation`은 컴파일과 실행 시 모두 필요할 때 사용
		- `testImplementation`은 테스트 시에만 필요할 때 사용

- **그래들 래퍼**
	- 프로젝트 루트에 포함된 `gradlew` 또는 `gradlew.bat` 파일
	- 그래들이 설치되어 있지 않아도 선언된 버전의 그래들을 자동으로 다운로드하여 실행
	- 팀원 모두가 동일한 빌드 환경을 유지 가능

- **유연한 스크립트**
	- Groovy나 Kotlin 언어를 기반으로 한 DSL(Domain Specific Language)을 사용하여 빌드 로직을 자유롭게 작성할 수 있다.

- **확장(Extensions)**
	- 플러그인이 제공하는 기능을 사용자 입맛에 맞게 설정할 수 있도록 빌드 스크립트에 노출된 설정 블록

### 성능 최적화 기술

그래들은 빌드 시간을 단축하기 위해 여러 지능적인 기능을 제공한다.

- **증분 빌드(Incremental Build)** 
	- 마지막 빌드 이후 변경된 파일이 있는 작업만 다시 실행하고, 나머지는 `UP-TO-DATE` 상태로 건너뛴다.

- **빌드 캐시(Build Cache)**
	- 이전 빌드의 결과물을 저장해 두었다가 입력값이 같을 때 다시 사용
	- 빌드 입력에 대한 해시를 기반으로 재사용 한다.

- **구성 캐시(Configuration Cache)**
	- 구성 단계의 결과물(작업 그래프)을 캐싱하여 빌드 시작 시간을 획기적으로 줄여줍니다.


### 빌드 생명주기 (Build Lifecycle)

-  **초기화(Initialization)** 
	- 빌드 대상 프로젝트를 결정하고 각 프로젝트에 대한 `Project` 인스턴스를 생성

- **구성(Configuration)** 
	- 빌드 스크립트를 실행하여 실행해야 할 작업(Task)들의 관계를 파악
	- 작업 그래프(Task Graph)를 생성

-  **실행(Execution)**
	- 구성 단계에서 선택된 작업들을 순서대로 실제 수행


### 주요 파일 

**[ 핵심 설정 파일 ]**

- **settings.gradle**
	- Gradle 빌드의 시작점
	- 빌드에 참여하는 프로젝트 구조와 서브프로젝트(모듈) 목록을 정의

- **build.gradle**
	- 개별 프로젝트(또는 서브프로젝트)의 실제 빌드 로직을 담고 있는 파일
	- 플러그인 적용, 저장소 설정, 의존성 선언, 태스크(Task) 정의 등이 이루어진다.

- **gradle.properties**
	- Gradle의 환경 설정 및 프로젝트 전역 속성을 관리
	- 예시
		- 병렬 빌드 활성화
		- 빌드 캐시 사용 여부
		- JVM 메모리 설정 (org.gradle.jvmargs)

**[ 그래들 래퍼 파일 ]**

Gradle 설치 없이도 동일한 버전으로 빌드를 실행할 수 있게 해주는 필수 파일들이다.

- **gradlew** **/** **gradlew.bat**
	- 유닉스(Linux/macOS) 및 윈도우용 실행 스크립트

- **gradle/wrapper/gradle-wrapper.properties**
	- 사용할 Gradle의 버전과 다운로드 경로 등 래퍼의 동작 방식을 설정

- **gradle/wrapper/gradle-wrapper.jar**
	- Gradle 배포판을 다운로드하는 실행 코드가 포함된 바이너리 파일

**[ 기타 ]**

- **gradle/libs.versions.toml** **(Version Catalog)**
	- 프로젝트 전체에서 사용하는 라이브러리와 플러그인의 버전을 한곳에서 관리

- **.gradle** **&** **build** **(디렉터리)**
	- Gradle이 빌드 과정에서 생성하는 **캐시와 결과물**이 저장


### Settings.gradle

```
// 1. 루트 프로젝트 이름 설정 (일관성을 위해 항상 지정하는 것이 권장됨)
rootProject.name = 'my-complex-project' [5, 6]

// 2. 빌드에 포함할 서브프로젝트(모듈) 정의
// 기본적으로 프로젝트 경로 ':app'은 물리적 디렉토리 './app'에 매핑된다.
include 'app', 'core', 'utils' [4, 7]
```
